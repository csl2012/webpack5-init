## vue2和vue3的区别

全量 TypeScript 重写、Monorepo 模块化架构、渲染层与平台解耦、响应式系统彻底重构（核心性能优化）Object.defineProperty → Proxy、组合式api、diff算法优化、支持 Fragment、Teleport、Suspense 等 Vue3 新特性、更好的 Tree-shaking 支持、静态提升（Static Hoisting）、PatchFlags、内联事件缓存（Event Cache）等

1. vue2使用Object.defineProperty，vue3使用Proxy实现响应式系统，性能更好，支持更多数据类型
2. vue2使用options api，vue3使用组合式api
3. vue3静态提升，patchFlags优化，性能更好
4. vue3支持TS，vue2不支持TS
5. vue3支持Fragment、Teleport、Suspense等新特性
6. diff算法：vue2使用双端比较，vue3使用最长递增子序列，性能更好
7. vue3 tree-shaking，支持按需引入，体积更小
8. Composition API（组合式 API）
这是 Vue3 最大的开发体验优化，彻底解决了 Vue2 Options API 的核心痛点。
Vue2 Options API 的痛点：
大型项目中，同一个功能的代码分散在 data、methods、computed、watch、生命周期里，维护难度极高
逻辑复用只能靠 mixin，存在命名冲突、来源不清晰、数据来源模糊的问题
对 TypeScript 支持极差，类型推导困难
Composition API 的优化收益：
逻辑内聚：同一个功能的代码可以写在一起，大型项目的可维护性大幅提升
逻辑复用：用组合式函数替代 mixin，没有命名冲突，数据来源清晰，复用性拉满
类型友好：纯函数式 API，天然支持完整的 TypeScript 类型推导
更好的代码压缩：函数名、变量名可以被更好的压缩，而 Options API 的对象属性名无法压缩
灵活的执行时机：可以在任何地方创建响应式数据，不限于组件内部，更灵活的逻辑组织
9. 组件 API 简化与统一
v-model 优化：支持组件上多个 v-model，无需像 Vue2 那样用.sync 修饰符 hack，同时简化了自定义 v-model 的实现，逻辑更清晰。
props/emits 标准化：新增defineProps/defineEmits编译器语法糖，原生支持类型推导，同时必须显式声明 emits 事件，解决了 Vue2 中事件来源模糊、难以维护的问题。
生命周期钩子优化：把生命周期钩子改成了onMounted/onUpdated等具名函数，和 Composition API 配合更友好，同时语义更清晰，避免了 Vue2 中 this 指向的问题。
移除不合理的 API：移除了$on/$off/$once事件总线（推荐用 mitt 替代）、移除了.native修饰符、移除了过滤器（推荐用计算属性替代），让 API 更简洁，减少了开发者的心智负担。

## vue2和vue3的响应式原理

vue2使用Object.defineProperty，vue3使用Proxy实现响应式系统，性能更好，支持更多数据类型
Vue2 的核心痛点：
只能监听对象已有属性的读 / 写，无法监听新增 / 删除属性，必须通过 Vue.set/Vue.delete hack 处理
数组只能通过重写 7 个变异方法实现监听，无法监听数组下标修改、长度修改
初始化时必须递归遍历整个对象的所有属性，大对象 / 大数据初始化性能极差
无法原生支持 Map、Set、WeakMap、WeakSet 等 ES6 集合类型
Vue3 的优化收益：
全属性监听：原生支持对象新增 / 删除属性、数组下标修改 / 长度修改，无需额外 API
懒监听：初始化时不需要递归遍历，只有属性被访问时才会做响应式代理，大对象初始化速度提升 50%+
原生支持集合类型：Map、Set、WeakMap、WeakSet 全量响应式支持
更细粒度的依赖收集：精准监听访问到的属性，避免不必要的更新
内存占用降低 50%：优化了 Dep/Watcher 依赖管理模型，减少了循环引用，内存泄漏风险大幅降低

## 事件循环

js是单线程的，一次只能做一件事，遇到同步代码直接执行，遇到异步代码，会先将异步代码放到事件队列中，等同步代码执行完毕后再执行异步代码。异步代码又分为宏任务和微任务，宏任务包括setTimeout‌、setInterval‌、setImmediate、I/O‌(例如，文件读取、网络请求等)、UI渲染‌等，微任务包括Promise.then/catch/finally‌、MutationObserver、process.nextTick、queueMicrotask‌等。先执行微任务，微任务执行完毕后再执行宏任务。

## 闭包

闭包是一个函数与其创建时所处的词法环境（lexical environment）的组合体。即使该函数在其原始作用域之外被调用，它仍能访问并“记住”其定义时所在作用域中的变量。常见的创建闭包的方式就是函数嵌套函数，返回内部函数。闭包的作用：封装私有变量，实现模块化，缓存变量，柯里化函数。闭包的缺点：内存泄漏，闭包会导致变量常驻内存，如果大量使用闭包，会导致内存占用过多，甚至内存泄漏。

## 原型和原型链

原型和原型链是 js 中的一个重要概念，它解释了 js 中对象的继承关系。每个对象都有一个原型对象，原型对象 again 也有一个原型对象，当查找对象的属性或方法时，会优先在对象自身查找，如果没有找到，会查找对象的原型，原型没有找到，会继续向上找原型的原型，以此类推，直到到达 null。这个过程就叫原型链。原型链的作用：
实现继承：通过原型链，子对象可以访问父对象的属性和方法。
实现共享：多个对象可以共享同一个原型对象的属性和方法，节省内存空间。
实现封装：通过原型链，可以将对象的私有属性和方法封装起来，对外提供公共的接口。
原型链的缺点：
原型链的查找过程是线性的，如果原型链过长，查找效率会降低。
原型链的修改会影响到所有继承自该原型的对象，可能会导致意外的副作用。
原型链的查找过程是线性的，如果原型链过长，查找效率会降低。

## 前端安全：XSS 和 CSRF 的防御

“XSS（跨站脚本攻击）：攻击者注入恶意脚本到网页中，窃取 Cookie 或执行恶意操作。
防御措施：
对用户输入进行转义（HTML、JS、CSS 编码）；
使用 CSP（内容安全策略）限制脚本来源；
Cookie 设置 HttpOnly 和 Secure 标志。

CSRF（跨站请求伪造）：攻击者诱导用户点击恶意链接，利用用户的登录态发送伪造请求。
防御措施：
使用 CSRF Token（每次请求携带随机 Token）；
验证 Origin / Referer 请求头；
Cookie 设置 SameSite=Strict 或 Lax；
重要操作需要二次验证（短信/验证码）。
React 对 XSS 有天然防护，因为 JSX 会转义输出内容，但 dangerouslySetInnerHTML 要谨慎使用。”

## 什么是跨域？CORS 的原理是什么？除了 CORS 还有哪些解决方案？

“CORS 原理：浏览器在发送跨域请求时，会自动添加 Origin 请求头。服务端返回 Access-Control-Allow-Origin 响应头，如果允许该源，浏览器才放行。
复杂请求（PUT/DELETE/自定义头）会先发送 OPTIONS 预检请求，确认服务端允许后才发送真实请求。

其他跨域方案：
JSONP：只支持 GET，利用 <script> 标签不受同源策略限制；
代理服务器：开发环境用 webpack devServer proxy，生产环境用 Nginx 反向代理；
postMessage：用于 iframe 跨域通信；
WebSocket：原生支持跨域；
document.domain：用于主域相同、子域不同的场景。”

## 用户需要上传一个 2GB 的视频文件，网络可能中断，如何实现断点续传？

文件切片：用 Blob.prototype.slice 将大文件切成 1MB-5MB 的切片；
生成唯一标识：用 SparkMD5 计算文件 hash，用于识别同一个文件；
断点记录：用 localStorage 或 IndexedDB 记录已上传的切片索引；
并发上传：用 Promise.all 控制并发数（比如同时上传 3 个切片）；
断点续传：上传前先向服务端查询已上传的切片列表，跳过已上传的切片；
切片合并：所有切片上传完成后，通知服务端合并。
还需要考虑：失败重试机制、进度条展示、暂停/恢复功能、网络恢复后自动续传。
